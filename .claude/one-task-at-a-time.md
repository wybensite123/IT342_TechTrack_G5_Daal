name: one-task-at-a-time
description: Standing Rule α — finish before proposing next, confirm before starting
type: feedback
---

One task at a time. Always.

- Finish the current task before proposing the next
- For any task requiring manual developer action (OAuth credentials, device, token paste): give exact numbered steps and wait for confirmation
- Never start a new task without explicit "go" from the developer
- Propose next candidates only after current task is done and Gate 3 (self-audit) passes

**Why:** Context switching mid-task creates half-done features that are harder to finish than to start.

**How to apply:** This is Standing Rule α. It applies all the time, every session, without exception.
