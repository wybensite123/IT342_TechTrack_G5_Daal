After any session of work:

- [ ] Update FEATURES.md checkboxes in `Docs/Skills/MASTER.md` Section 14 — check off completed items
- [ ] Note any new architectural decision with reasoning (the WHY, not just the WHAT)
- [ ] Verify `git push` succeeded
- [ ] Report specifically: which files changed, which commits were created, what changed

**Why:** Progress tracking lives in MASTER.md. If not updated, the next session has no reliable feature state.

**How to apply:** Always run this before ending a session. Combine with Gate 6 (session-handoff).
